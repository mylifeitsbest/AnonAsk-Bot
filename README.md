# 🕵️‍♂️ AnonAsk Bot


## 🔥 Киллер-фичи 

* **Режим «Тоталитаризм»**: Если твой Telegram ID внесен в список админов, анонимность ломается. Ты будешь видеть юзернеймы тех, кто тебе пишет. Контролируй входящий трафик.
* **Потоковый клининг-сервис**: Активные диалоги живут ровно 24 часа. Бот запускает отдельный независимый поток (`threading`), который каждый час автоматически подчищает оперативную память от просроченных сессий.
* **Удобные инлайн-ответы**: Получателю не нужно гадать, как ответить анониму. Под сообщением генерируется умная кнопка `✉️ Ответить`, которая мгновенно связывает диалог в обратную сторону.
* **Красивое консольное логирование**: Каждое действие (старт, отправка сообщения, создание ссылки) фиксируется в консоли с точным таймстампом.

## 🛠 Стек технологий

* **Язык**: Python 3.x
* **Библиотека**: `pyTelegramBotAPI` (`telebot`)
* **Компоненты**: `threading` (для параллельного таск-менеджера), `datetime` (управление сессиями).

## 🚀 Быстрый старт

### 1. Установка зависимостей
Убедись, что у тебя установлена библиотека для работы с API Telegram. Если нет — запускай в терминале:
```bash
pip install pytelegrambotapi
```

### 2. Настройка конфигурации
Открой файл с кодом и замени значения на свои:
```python
API_TOKEN = 'ТВОЙ_ТОКЕН_ОТ_BOTFATHER'
ADMIN_IDS = [123456789]  # Сюда вставь свой ID, чтобы деанонить хейтеров
```

### 3. Запуск
```bash
python main.py
```

## 📋 Архитектура работы (Под капотом)

1. **Юзер А** нажимает `/start` без параметров $\rightarrow$ Бот генерирует его уникальную дипплинк-ссылку формата `t.me/bot?start=ID`.
2. **Юзер Б** переходит по ссылке $\rightarrow$ Бот связывает их ID в словарь `active_conversations` на 24 часа и ждет текст.
3. Любое сообщение от **Юзера Б** пересылается **Юзеру А** с инлайн-кнопкой, внутри которой зашит callback-дата с ID отправителя.
